# OmniKernel App

Runnable example app that exposes the `@nestjs-yalc/omnikernel-module` through
both GraphQL and REST.

## What it demonstrates

- `OmniKernelModule.register('default')` as the shared provider layer
- GraphQL resolvers generated by `CrudGenDependencyFactory`
- REST endpoints generated by `crudRestControllerFactory`
- one SQLite in-memory app surface where REST and GraphQL operate on the same
  underlying OmniKernel services

## Exposed REST resources

- `GET/POST/PUT/DELETE /omni/named`
- `GET/POST/PUT/DELETE /omni/records`
- `GET/POST/PUT/DELETE /omni/documents`
- `GET/POST/PUT/DELETE /omni/collections`
- `GET/POST/PUT/DELETE /omni/relations`
- `GET/POST/PUT/DELETE /omni/external-refs`

All REST controllers are thin wrappers generated by
`crudRestControllerFactory`, so they reuse the same `GenericService` instances
that power the module's GraphQL resolvers.

## Exposed GraphQL

- `/graphql`
- auto-generated OmniKernel CRUD queries and mutations from the module

## Run

```bash
npm run test:e2e --prefix examples/omnikernel-app
```
